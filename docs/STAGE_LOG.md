# Stage breakthrough log

This append-only log answers three questions for every bounded breakthrough:
**when did it happen, what changed, and what effect was demonstrated?** It is a
navigation aid, not a substitute for raw test reports, GitHub Actions, or design
records.

## Recording rule

Add an entry during the same working session when a small stage first becomes
demonstrable, rejects a known failure, or is promoted to a stronger validation
boundary. Every entry must include:

- an ISO 8601 timestamp with timezone;
- the bounded stage and whether evidence is local, hosted, or protected-main;
- the action taken and the observed effect, with measured values when available;
- a link to durable evidence or an explicit limitation.

Frontend or interaction stages also require a screenshot of the exact state that
was exercised. Prefer a hosted Playwright attachment. A screenshot illustrates
the state but never replaces semantic assertions, accessibility checks, or event
truth. If endpoint protection rewrites local images, show an in-memory browser
screenshot in the working task and require an unmodified hosted artifact before
promotion.
